# Contributing to Nexus Simulator

Thank you for your interest in contributing to the Nexus Simulator project! We welcome contributions from everyone.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on GitHub. Please include:
- A clear description of the bug.
- Steps to reproduce the behavior.
- Expected and actual behavior.
- Any relevant logs or screenshots.

### Suggesting Features

We welcome feature requests! Please open an issue and describe the feature you'd like to see, why it's needed, and how it should work.

### Pull Requests

1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Make your changes.
4. Ensure your code follows the coding style of the project.
5. Submit a pull request against the `main` branch.

## Coding Standards

- **Backend (C++)**: Follow standard OpenThread coding conventions where applicable.
- **Frontend (TS/React)**: Use TypeScript, follow React best practices, and use pre-defined components and design tokens.
